# 毎朝天気予報 Discord 通知（Google Apps Script）

気象庁の天気予報データを使って、毎朝自動で Discord にその日の天気を投稿する Google Apps Script です。

外部 API キーや有料サービスは不要。Discord の Webhook と Google Apps Script だけで動きます。

## 機能

- 毎朝 **6 時台後半（だいたい 6:30〜7:00 ごろ）**に、指定した Discord チャンネルへ自動投稿（通常投稿）
- 気象庁の **11時発表・17時発表**で天気予報に大きな変化があった場合のみ**追加通知**
- 気象庁の JSON データを使用（無料・API キー不要）
- 表示する天気情報：

| 情報 | 説明 |
|------|------|
| 日付・曜日 | 投稿実行日の日付と曜日 |
| 地域名 | 設定した地域名 |
| 天気 | 今日の天気概況（気象庁発表） |
| 降水確率 | 時間帯別（午前/午後/夜/未明） |
| 雨のピーク | 降水確率が最も高い時間帯（確率ベースの推定） |
| 最高・最低気温 | 予想気温（値が同じでも「最高 25℃ / 最低 25℃」のように表示。片方のみの場合は「最高 28℃」のように取得できた方だけ表示） |
| 風 | 風向・風速の概況 |
| 服装目安 | 気温・降水確率・風からの簡易推定 |

> **注意**: 湿度・UV インデックスは気象庁の無料 JSON だけでは取得できないため、本スクリプトでは表示していません。

## 必要なもの

- Google アカウント（個人の Gmail アカウントで OK）
- Discord サーバーの「ウェブフックを作成」権限

## セットアップ手順

### 1. Discord Webhook を作成

1. Discord で通知を送りたいチャンネルの **⚙ チャンネル設定** を開く
2. **連携サービス** → **ウェブフック** を開く
3. **新規ウェブフック** をクリック
4. 名前を適当に設定（例: `天気予報`）
5. **ウェブフック URL をコピー** をクリックして URL を控える

> **重要**: この URL は誰でもそのチャンネルにメッセージを投稿できる秘密情報です。**絶対に GitHub や SNS に公開しないでください。**

### 2. Google Apps Script プロジェクトを作成

1. [Google Apps Script](https://script.google.com/) にアクセス
2. **新しいプロジェクト** をクリック
3. 左上の「無題のプロジェクト」をクリックして、好きな名前を付ける（例: `天気予報Discord通知`）

### 3. Code.gs を貼り付ける

1. このリポジトリの [`Code.gs`](./Code.gs) を開き、**中身をすべてコピー**
2. GAS エディタの `コード.gs`（最初からある空のファイル）に **すべて貼り付けて上書き**
3. 必要に応じて、ファイル上部の `CONFIG` を編集（→ [地域設定の変え方](#地域設定の変え方)）

### 4. Webhook URL を設定

Webhook URL はコードに直接書かず、GAS の**スクリプトプロパティ**に保存します。

1. GAS エディタ左メニューの **プロジェクトの設定 ⚙** をクリック
2. 一番下の **スクリプト プロパティ** セクションを開く
3. **行を追加** をクリック
4. 以下のように入力：

   | プロパティ | 値 |
   |---|---|
   | `DISCORD_WEBHOOK_URL` | Discord でコピーした Webhook URL |

5. **スクリプト プロパティを保存** をクリック

### 5. テスト投稿

1. GAS エディタ上部の関数選択ドロップダウンで **`testPost`** を選択
2. **▷ 実行** をクリック
3. 初回実行時は権限の承認画面が出ます。「権限を確認」→ 自分のアカウントを選択 → 「詳細」→ 「（プロジェクト名）に移動」→ 「許可」
4. Discord チャンネルに天気予報が投稿されていれば成功です

> `testPost` は固定のサンプルメッセージではなく、実際に気象庁 JSON を取得して投稿します。本番と同じパスを通るため、設定確認に最適です。
> Discord Webhook にはレート制限があるため、`testPost` を短時間に連続実行しないでください。

更新確認（11時・17時発表）の動作テストには **`testUpdateCheck`** を使ってください。前回保存したスナップショットと現在の予報を比較し、変化の有無をログで確認できます（実際の投稿は行いません）。

### 6. 自動実行トリガーを設定

1. GAS エディタ上部の関数選択で **`setupTrigger`** を選択
2. **▷ 実行** をクリック
3. 以下の 3 つのトリガーが作成されます：

   | トリガー | 実行時間 | 機能 |
   |----------|----------|------|
   | `main()` | 毎日 6:30〜7:00 ごろ | **通常投稿**（毎回投稿） |
   | `checkUpdate()` | 毎日 11:00〜11:30 ごろ | **11時発表の更新確認**（変化時のみ追加通知） |
   | `checkUpdate()` | 毎日 17:00〜17:30 ごろ | **17時発表の更新確認**（変化時のみ追加通知） |

トリガーの確認・削除は、GAS エディタ左メニューの **トリガー 🕓** から行えます。

> **補足**: GAS の `nearMinute` は厳密な時刻指定ではありません。朝の通常投稿は 6:45 前後（だいたい 6:30〜7:00 ごろ）、更新確認は 11:15・17:15 前後に実行され、日によって多少前後します。

## 地域設定の変え方

`Code.gs` の冒頭にある `CONFIG` オブジェクトを編集します。

```javascript
var CONFIG = {
  AREA_CODE: '110000',                          // ← 都道府県のエリアコード
  REGION_CODE: '110010',                        // ← 予報区域コード（class10s）
  TEMP_STATION_CODE: '43241',                   // ← 気温観測地点コード（class20s）
  REGION_NAME: '埼玉県南部（春日部・吉川周辺）', // ← Discord に表示する地域名
  MORNING_TRIGGER_HOUR: 6,                      // ← 通常投稿の時間
  MORNING_TRIGGER_MINUTE: 45,                   // ← 通常投稿の分（前後約15分）
  UPDATE_CHECK_HOURS: [11, 17],                 // ← 更新確認の時間
  UPDATE_CHECK_MINUTE: 15,                      // ← 更新確認の分（前後約15分）
  POP_CHANGE_THRESHOLD: 30,                     // ← 降水確率変化のしきい値（%）
  TEMP_CHANGE_THRESHOLD: 3,                     // ← 気温変化のしきい値（℃）
  // ...
};
```

### 自分の地域のコードを調べる

気象庁のエリア定義 JSON から、自分の地域のコードを調べられます：

**[https://www.jma.go.jp/bosai/common/const/area.json](https://www.jma.go.jp/bosai/common/const/area.json)**

この JSON の構造：

| キー | 内容 | 例（埼玉） |
|------|------|-----------|
| `offices` | 都道府県単位の気象台情報 | `"110000"` → 埼玉県（熊谷地方気象台） |
| `class10s` | 予報区域（天気・風・降水確率に使用） | `"110010"` → 南部, `"110020"` → 北部 |
| `class15s` | より細かい区域 | `"110011"` → 南中部 |
| `class20s` | 気温観測地点 | `"43241"` → さいたま |

| 沖縄県 | `471000` | 沖縄気象台 |

### 地域名の表示方針（予報区域ベース）

気象庁の天気予報は「市区町村」単位ではなく「予報区域（class10s）」単位で発表されます。たとえば埼玉県南部（コード `110010`）には、さいたま市、春日部市、吉川市、川口市など多くの市町村が含まれます。

同じ予報区域内の市町村は、天気・降水確率・風の予報が同一です。そのため、別々に表示する必要はなく、**1 つの予報区域としてまとめて表示**する方針を推奨します。

`REGION_NAME` には次のような形式を推奨します：

```
予報区域名（主な対象市町村）
```

例：
- `埼玉県南部（春日部・吉川周辺）` ← 初期設定
- `埼玉県南部（さいたま市周辺）`
- `東京都東京地方（23区）`

気温のみ地点コード（`TEMP_STATION_CODE`、class20s）で取得されます。各市町村がどの予報区域に属するかは [area.json](https://www.jma.go.jp/bosai/common/const/area.json) で確認してください。

#### 春日部市・吉川市の方向け

春日部市も吉川市も埼玉県南部（予報区域コード `110010`）に属するため、天気・降水確率・風の予報内容は同じです。このスクリプトの初期設定では、両市をまとめて `埼玉県南部（春日部・吉川周辺）` と表示します。

気温観測地点には `43241`（さいたま）を使用しています。春日部・吉川に最も近い観測地点ではないものの、同じ南部区域内で大きな差はないため実用上問題ありません。

## 投稿文のカスタマイズ

### 気象庁の出典を表示する

`CONFIG` の `SHOW_SOURCE_IN_DISCORD` を `true` に変更すると、投稿文の末尾に気象庁の出典が表示されます。

```javascript
SHOW_SOURCE_IN_DISCORD: true,  // Discord 投稿の末尾に出典を表示
```

**初期値は `false` です。** 気象庁のデータを利用していることをユーザーに明示したい場合は `true` にしてください。

### 服装目安の閾値を変えたい

服装目安のロジックは `getClothingAdvice()` 関数で定義されています。気温の閾値やアドバイス文言を必要に応じて編集してください。

### 更新通知のしきい値を変えたい

「大きな変化」とみなす条件は `CONFIG` の以下の値で調整できます：

```javascript
POP_CHANGE_THRESHOLD: 30,   // 降水確率の変化（%）がこれ以上で通知
TEMP_CHANGE_THRESHOLD: 3,   // 気温の変化（℃）がこれ以上で通知
```

変化検出のロジック自体を変更したい場合は `detectSignificantChanges()` 関数を編集してください。

## 更新通知の仕組み

本スクリプトは、気象庁の **5時・11時・17時** の1日3回の発表に対応しています。

### 朝6時台の通常投稿

- 毎朝 6:30〜7:00 ごろに `main()` が実行され、**大きな変化の有無に関係なく必ず投稿対象**になります
- 基本的には気象庁の 5時発表の予報が使われます
- Discord への投稿処理後、天気情報のスナップショットをスクリプトプロパティに保存します

### 11時台・17時台の更新確認

- 11:15・17:15 前後（それぞれ前後約15分）に `checkUpdate()` が実行されます
- 気象庁 JSON を取得し、前回投稿時のスナップショットと比較します
- **大きな変化があった場合のみ** Discord に追加通知します
- 変化が小さい場合は投稿しません（無駄な通知を防ぎます）
- **重要**: 大きな変化の有無に関わらず、JSONから今日の気温が取得できた場合は**必ず気温スナップショットを保存**します。これにより、後続の発表（例: 17時発表）でJSONに今日の気温が含まれない場合でも、11時台に取得した気温を補完に使えます

### 「大きな変化」として扱う条件

| 条件 | しきい値 | 設定箇所 |
|------|----------|----------|
| 天気が「晴れ・くもり系」から「雨・雪系」に変わった | — | `detectSignificantChanges()` |
| 降水確率の最大値が変化 | 30% 以上 | `POP_CHANGE_THRESHOLD` |
| 雨のピーク時間帯が変わった | — | `detectSignificantChanges()` |
| 風が強くなった（例: 「やや強く」が追加） | — | `detectSignificantChanges()` |
| 最高/最低気温が変化 | 3℃ 以上 | `TEMP_CHANGE_THRESHOLD` |
| 服装目安の内容が大きく変わった | — | `detectClothingChange()` |

### スナップショットの保存先

- 比較用の天気スナップショット: GAS の**スクリプトプロパティ**の `WEATHER_SNAPSHOT` キー
- 気温スナップショット（日付別）: **スクリプトプロパティ**の `WEATHER_TEMP_SNAPSHOT_YYYY-MM-DD` キー（例: `WEATHER_TEMP_SNAPSHOT_2026-06-27`）
- 気温スナップショットは日付キーで管理されるため、昨日の気温が今日の投稿に使われることはありません
- 保存状況の確認には `debugTemperatureSnapshot()` を実行してください

## 投稿文の例

### 朝6時台の通常投稿

```
2026年6月27日（土）の天気

地域：埼玉県南部（春日部・吉川周辺）
天気：雨 時々 くもり
降水確率：午後 60% / 夜 60%
雨のピーク：午後・夜が雨のピーク（降水確率 60%）
気温：最高 25℃ / 最低 18℃
風：南東の風 後 北の風 やや強く

服装目安：半袖中心でよさそうです。雨の可能性があるので、折りたたみ傘があると安心です。風が強いので、風を通しにくい上着がおすすめです。
```

### 11時・17時発表の更新通知（変化があった場合のみ）

```
## 天気予報の更新

地域：埼玉県南部（春日部・吉川周辺）
変更点：降水確率が30%上昇しました（最大60%）。雨のピーク時間帯が「午後・夜」に変わりました。

天気：雨 時々 くもり
降水確率：午後 60% / 夜 60%
雨のピーク：午後・夜
気温：最高 25℃ / 最低 18℃
風：南東の風 後 北の風 やや強く
```

- 発表時刻（「発表：2026年6月27日 11:00（熊谷地方気象台）」）は Discord 投稿文には**表示しません**。内部ログには出力されます
- 「おはようございます」などの挨拶文は意図的に**入れていません**
- 最高気温と最低気温が同じ値でも **「最高 25℃ / 最低 25℃」** とそのまま表示します。「25℃前後」にはなりません
- **夕方の投稿と気温表示について**: 気象庁の17時発表のJSONでは、今日の気温データが含まれず、明日以降の日付のみになることがあります。その場合でも、同日中の先の発表（朝6時台や **11時台**）で保存した気温スナップショットから補完して表示します。11時台は投稿しなかった場合でも気温データを保存するため、17時台の補完に使えます。気温行が省略されるのは、スナップショットも存在しない初回実行時など、本当にデータがない場合だけです
- 降水確率は、夕方の投稿では残り時間帯のみ（例: 「夜」のみ）表示されることがあります。これは気象庁JSONの降水確率データが現在時刻以降の時間帯だけを含むためです。気温とは異なり、降水確率はスナップショットから補完しません（最新の予報を使うため）
- 取得できなかった項目（例: 風データがない場合）は自然に**省略**されます
- 「不明」と表示されることはありません

## トラブルシューティング

### Webhook URL が設定されていません

GAS の実行ログにエラーが出る場合、スクリプトプロパティに `DISCORD_WEBHOOK_URL` が正しく設定されているか確認してください。

### Discord 投稿で 429 / error code: 1015 が出る

Discord Webhook と、その手前の Cloudflare にはレート制限があります。短時間に多数の投稿を行うと、HTTP 429 または `error code: 1015` が返ることがあります。

- `testPost` を短時間に連続実行しないでください
- 通常実行で 429 / 1015 が返った場合、投稿本文をスクリプトプロパティに一時保存し、10〜20分後に自動で**1回だけ**再投稿します
- 再投稿もレート制限された場合は無限に繰り返さず終了し、保留データをスクリプトプロパティに残します

### トリガーが動かない

- GAS エディタ左メニューの **トリガー** で、3つのトリガー（`main`、`checkUpdate` ×2）が有効か確認
- トリガーが足りない場合は、`setupTrigger()` を再実行すると既存トリガーを削除してから作り直します
- **実行数** に失敗が記録されている場合、クリックするとエラー内容が表示されます
- GAS の実行は Google アカウントあたりの**日次割り当て**があります（通常の個人利用ではまず上限に達しません）

### 「このアプリは Google で確認されていません」

初回実行時にこの警告が出ます。自分で作成したスクリプトであることを確認し、「詳細」→「（プロジェクト名）に移動」→「許可」で進めてください。

## 注意事項・免責

### 秘密情報の取り扱い

- **Discord Webhook URL をコードや README に直書きしないでください**
- Webhook URL は必ず GAS のスクリプトプロパティで管理してください
- 誤って GitHub に Webhook URL をプッシュした場合、Discord 側で Webhook を削除・再作成してください
- `.env` ファイルや `secrets.*` ファイルは `.gitignore` で除外されています

### 気象庁 JSON の仕様変更について

- 本スクリプトが使用している気象庁の JSON データ（`https://www.jma.go.jp/bosai/forecast/`）は、**公式に安定保証された API ではありません**
- 気象庁のウェブサイト改修などにより、JSON の URL や構造が**予告なく変更される**可能性があります
- その場合、スクリプトが動かなくなるため、コードの修正が必要です
- 変更を検知した場合は、GitHub の Issues などでお知らせください

### 服装目安について

服装の目安は**気温・天気・降水確率・風からの簡易的な推定**であり、実際の体感とは異なる場合があります。あくまで参考情報としてご利用ください。

### 雨のピーク時間帯について

降水確率ベースの推定であり、実際の降水量（mm/h）とは異なります。気象庁の無料 JSON では時間帯別の降水量が提供されていないため、降水確率を代替指標として使用しています。

### 気象庁データの利用

このスクリプトは**[気象庁](https://www.jma.go.jp/bosai/forecast/) の天気予報データ**を利用しています。

- データ提供: 気象庁 (Japan Meteorological Agency)
- 取得先 URL: `https://www.jma.go.jp/bosai/forecast/data/forecast/`
- データは JSON 形式で取得・加工しています
- 本スクリプトは気象庁の公式ツールではありません

## clasp を使う場合（上級者向け）

通常は GAS エディタにコードを貼り付けるだけで使えますが、[clasp](https://github.com/google/clasp) を使ってローカル開発・Git 管理したい場合は以下の手順で行えます。

### clasp のセットアップ

```bash
# clasp のインストール
npm install -g @google/clasp

# Google アカウントにログイン
clasp login

# GAS プロジェクトをクローン（既存プロジェクトがある場合）
clasp clone <スクリプトID>

# または新規プロジェクト作成
clasp create --type standalone --title "天気予報Discord通知"
```

### clasp 運用時の注意

- `.clasp.json` はプロジェクト設定ファイルのため Git 管理しても問題ありません
- 一方、**認証情報ファイル（`~/.clasprc.json` など）は絶対にコミットしないでください**
- `.gitignore` にあらかじめ認証ファイルが指定されています
- スクリプトプロパティ（`DISCORD_WEBHOOK_URL`）やトリガー設定は `clasp push` ではデプロイされません。GAS エディタ上で手動設定するか、`clasp open` でエディタを開き、スクリプトプロパティの設定と `setupTrigger()` の実行を行ってください
